Final Project

Store all of your files for this lab in a folder named "final".

Create a poker solitaire game.

It is a single-player game played by dealing cards, one at a time, onto a 5x5 grid. When the grid is filled, your score will be the sum of the scores of the rows and columns. The score of a row or column depends on the poker hand that can be created from the 5 cards of that row/column. Once a card has been placed onto the grid, it cannot be moved.

Here are the poker hand types and the number of points they are worth:

  • Royal Flush (100 points): An ace-high straight flush (Ace, King, Queen, Jack, 10 all of the same suit)
  • Straight Flush (75 points): 5 cards of sequential rank, all of the same suit
  • Four of a Kind (50 points): 4 cards of the same rank and 1 card of another rank
  • Full House (25 points): 3 cards of the same rank and 2 cards of another 2nd rank.
  • Flush (20 points): 5 cards of the same suit, not all of sequential rank
  • Straight (15 points): 5 cards of sequential rank, not all of the same suit
  • Three of a Kind (10 points): 3 cards of the same rank, 2 cards of 2 other different ranks
  • Two pair (5 points): 2 cards of one rank, 2 cards of another 2nd rank, and 1 card of a 3rd rank
  • One pair (2 points): 2 cards of one rank, and 3 cards of 3 other different ranks

Try playing poker solitaire online here to get an idea of how it works.

Your program should meet these specifications:

  • Before the game begins, the program will ask the user for her name.
  • A shuffled, 52-card deck should be used within the program
  • Before each deal, the total score of the current user should be displayed and the grid should be displayed with it's rows and columns labeled.
  • The cards on the grid should be shown as abbreviations ("KD" for King of Diamonds, "10H" for 10 of Hearts, "2C" for 2 of Clubs, etc.)
  • The user should specify the coordinate of where to place the dealt card
  • The current score should be updated when an entire row or column as been filled.
  • At the end of the game, the top 3 high scores should be displayed as well as the users who attained those scores

You may name your source file(s) whatever you like, place the compiler command that you use into a script file named "build.sh"

An sh file contains Bash commands. Executing an sh file will execute all the Bash commands in that file, line by line. Here is an example of a possible build.sh script for this assignment:

#!/bin/bash g++-5 -std=c++14 main.cpp Deck.cpp

The above build script will compile main.cpp and Deck.cpp to produce an executable file named a.out. To execute build.sh, it must be marked as executable itself. After creating build.sh, use the following command to mark build.sh as executable:

chmod +x build.sh

Now you may run build.sh in the current directory with the ./build.sh command. Everytime you edit your source files, you may run ./build.sh to recompile your source files. Having a build script not only relieves you of having to memorize or re-type the compilation command(s), but it makes it easier for other people to compile your project.

  • High scores should be stored in external files which can be loaded or saved using fstream
  • A 2-dimensional vector or array can be used to store the 5x5 grid of cards
